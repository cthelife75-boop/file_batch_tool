# 安全政策

## 报告安全漏洞

我们非常重视项目的安全性。如果您发现了安全漏洞，请按照以下步骤报告：

### 报告方式

请通过以下方式报告安全问题：

- 发送电子邮件至：[3331648097@qq.com]
- 主题请注明：[Security Vulnerability] File Batch Tool

### 报告内容

请在报告中包含以下信息：

1. 漏洞的简要描述
2. 复现步骤（如果可能）
3. 受影响的版本
4. 可能的影响（数据泄露、拒绝服务等）
5. 任何修复建议（可选）

### 响应时间

我们会在收到报告后的 48 小时内确认收到，并在合理时间内提供修复方案。

### 公开披露

我们要求在我们修复漏洞并发布安全更新之前，不要公开披露漏洞细节。

## 支持的版本

我们会为以下版本提供安全更新：

| 版本 | 支持状态 |
|------|---------|
| 1.2.x | ✅ 完全支持 |
| 1.1.x | ✅ 安全修复 |
| 1.0.x | ❌ 不再支持 |

## 安全最佳实践

使用本项目时，请遵循以下安全最佳实践：

1. **备份重要文件**：在批量操作前，请备份重要文件
2. **谨慎使用**：理解每个操作的影响后再执行
3. **权限管理**：确保程序只拥有必要的文件读写权限
4. **及时更新**：使用最新版本以获得安全修复
5. **病毒扫描**：处理下载的文件前请进行病毒扫描

## 安全功能

本项目包含以下安全功能：

- ✅ 不会自动删除文件（除非明确操作）
- ✅ 支持预览操作结果（GUI 模式）
- ✅ 提供详细的操作日志
- ✅ 错误处理机制防止数据丢失

## 已知限制

- 本工具主要用于文件管理，不涉及网络通信（除 AI 功能外）
- AI 功能需要用户提供 API Key，我们不会存储或传输您的密钥
- 程序运行在您的本地环境，不会向任何第三方服务器发送数据
